# Подключение проектных правил Swipey для команды

## Что уже находится в репозитории

- `PROJECT_GUIDE.md` — подробное фактическое описание приложения;
- `README.md` — краткий обзор и продуктовый roadmap;
- `AGENTS.md` — обязательные правила для AI-агентов в этом репозитории;
- `.agents/skills/maintain-project-docs/` — проектный Codex skill, синхронизирующий документацию с кодом.

После коммита этих файлов коллегам достаточно получить актуальную версию репозитория. Skill не содержит абсолютных пользовательских путей и работает относительно корня проекта.

## Рекомендуемая установка

1. Закоммитить `.agents/skills/maintain-project-docs`, `AGENTS.md`, `PROJECT_GUIDE.md` и этот файл.
2. Коллеге выполнить обычный `git pull` или заново клонировать репозиторий.
3. Открыть корень `Swipey` как workspace в Codex.
4. Начать новую задачу, чтобы Codex заново обнаружил `AGENTS.md` и список project-local skills.
5. При необходимости вызвать skill явно: `$maintain-project-docs`.

Пример запроса:

```text
Используй $maintain-project-docs и добавь фильтрацию медиатеки по фото и видео.
```

Обычно явный вызов не нужен: `AGENTS.md` требует использовать skill для каждой задачи в репозитории.

## Если project-local skills не обнаруживаются

Можно установить копию skill в пользовательский каталог Codex:

```bash
mkdir -p ~/.codex/skills
cp -R .agents/skills/maintain-project-docs ~/.codex/skills/
```

После копирования следует перезапустить Codex или открыть новую задачу. Локальную копию потребуется обновлять вручную после изменений skill, поэтому предпочтительный источник — версия внутри репозитория.

## Проверка установки

Попросите Codex выполнить небольшую задачу в Swipey. В начале работы он должен сообщить, что использует `maintain-project-docs`, прочитать `PROJECT_GUIDE.md`, а в конце:

- обновить документацию, если факты проекта изменились;
- либо явно сказать, что документация проверена и обновление не требуется.

Материальные изменения должны появляться в разделе `Documentation change log` файла `PROJECT_GUIDE.md`.

## Для других AI-инструментов

Даже если инструмент не поддерживает Codex skills, он обычно сможет прочитать корневой `AGENTS.md`. Если не сможет, добавьте в командные инструкции инструмента правило: перед работой читать `AGENTS.md` и `PROJECT_GUIDE.md`, а после изменений синхронизировать документацию.
